# سوف 360 — المنصة السياحية لولاية الوادي

منصة ويب لاستكشاف المعالم السياحية والتراثية لولاية الوادي عبر خريطة تفاعلية،
مبنية بـ Next.js 16 (App Router) + React 19 + Supabase + Leaflet.

## البنية العامة

| المسار | الوصف |
|---|---|
| `/` | صفحة الدخول الرئيسية (توجيه إلى الخريطة أو الاستكشاف) |
| `/map` | الخريطة التفاعلية الكاملة (بحث، فلترة، ملاحة GPS، مسارات حقيقية) |
| `/explore` | صفحة استكشاف غنية (المعالم، ذكريات قديمة، تجارب الزوار المعتمدة) |
| `/about` | من نحن |
| `/privacy` | سياسة الخصوصية |
| `/admin` | لوحة تحكم إدارية محمية (راجع `ADMIN_DASHBOARD.md`) |

## التشغيل محلياً

```bash
npm install
cp .env.example .env.local   # ثم املأ القيم من لوحة Supabase
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000).

## متغيرات البيئة

راجع `.env.example` للقائمة الكاملة. باختصار:
- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` — إلزاميان لعمل الموقع.
- `SUPABASE_SERVICE_ROLE_KEY` — إلزامي لعمل لوحة التحكم الإدارية (سرّي، سيرفر فقط).
- `OSRM_SERVER_URL` — اختياري، لحساب مسارات ملاحة عبر خادم OSRM خاص بك.

## قاعدة البيانات

نفّذ `supabase/admin_schema.sql` كاملاً في Supabase SQL Editor لإنشاء الجداول
والصلاحيات المطلوبة (`admin_profiles`, `site_settings`, أعمدة `places`, وسياسات RLS).

## التوثيق الإضافي

- `ADMIN_DASHBOARD.md` — تفاصيل لوحة التحكم وخطوات إنشاء أول حساب مدير.
- `AUDIT_REPORT_AR.md` — سجلّ التدقيق الفني للمشروع وآخر الإصلاحات المطبّقة.
- `AGENTS.md` / `CLAUDE.md` — تعليمات لأدوات البرمجة بالذكاء الاصطناعي.
